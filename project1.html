Seminar Record Management System

Course: Data Structures and Algorithms
Type: Partner Project
Timeline: Fall, 2024

Project Overview

Implemented a comprehensive memory management system for storing and managing variable-length seminar records, integrating both a memory pool manager and a hash table for efficient record retrieval. The system was designed to handle dynamic memory allocation with a first-fit approach and extensible hash table implementation. This project required managing serialized seminar records containing multiple fields including title, date/time, length, keywords, location coordinates, description, cost, and unique ID.

Technical Implementation

  • Data Structures Used:
    • Hash Table with quadratic probing for collision resolution
    • Doubly linked list for managing free memory blocks
    • Byte array for the memory pool
    • Record serialization system for seminar data
  • Algorithms Implemented:
    • Hash function: h(k) = k mod M (where M is table size)
